# ADR-001: 选择 Qwen2.5-0.5B 作为代理模型

## Status

**Final**（实验已终止，结论见 docs/experiment-report.md）

## Context

本工具需要一个小型语言模型作为"代理"来扫描 Prompt 文本的物理信号（注意力熵、隐藏态范数）。候选方案：

1. **Qwen2.5-0.5B**（最终选择）— 仅 0.5B 参数，约 1GB 内存，Mac M 可运行
2. **Llama-3.2-1B** — 1B 参数，约 2GB 内存，表达能力略强但更重
3. **直接使用生产环境 70B 模型** — Attention 分布最精确，但显存需求 8×A100，完全不可行
4. **纯规则启发式**（无模型）— 无依赖，但无法捕捉 Transformer 内部的表征特征

选择 0.5B 的理论依据：利用 Transformer **浅层（Layer 1-4）句法同构性**。小模型和大模型在浅层的注意力模式高度相似，小模型的"高熵"位置大概率也是大模型的"高熵"位置。我们放弃高层语义推理，只做底层信号体检。

## Decision

选择 **Qwen2.5-0.5B**，原因：

- **同源性**：与生产环境主流大模型同属 Qwen 系列，Tokenizer 100% 对齐，确保 Token 边界一致
- **资源需求**：约 1GB 内存，纯 CPU 可运行，Mac M 系列无需 GPU
- **速度**：0.5B 推理延迟低，适合交互式分析
- **社区生态**：HuggingFace 下载量高，文档完善，问题修复活跃

## Consequences

- **更好**：低成本运行，人人可本地部署，无需 GPU 预算
- **更好**：推理速度快，用户体验流畅
- **更差**：0.5B 的 Attention 分布不完全等同于 70B，存在测量误差
- **更差**：无法检测仅在大模型深层才出现的语义混淆问题
- **需要接受**："0.5B ≠ 70B" 必须在 UI 和文档中明确标注，防止用户过度解读结果
- **实验验证**：实际测试证实 0.5B 的浅层 attention 无法可靠区分"逻辑谬误"与"复杂指令"。详见 `docs/experiment-report.md`
