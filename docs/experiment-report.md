# /reconcile —— 「注意力 Linter」实验漂移报告
**报告编号**：EXP-2026-0624-ATTN-LINTER  
**实验主体**：Reasonix & Agent Engineer (1.5 小时联合实测)  
**结论状态**：**方向性终止（Terminated with Extreme Prejudice）**

### 1. PRD 愿景（承诺的"应该"）
> "基于同源小模型（0.5B）的浅层物理量（熵变率与范数），构建一款开箱即用的 Prompt CT 扫描仪。用户输入任意 Prompt，无需调参，系统自动高亮'结构脆弱性'与'表征坍塌'。"

### 2. 实验事实（实测的"现实"）

| 核心指标 | 理论/预期行为 | 实际观测结果 | 漂移定性 |
| :--- | :--- | :--- | :--- |
| **熵变率 (Delta)** | 坏 Prompt（逻辑混乱） >> 好 Prompt（逻辑清晰） | 坏文本 0.96，但 **"写快排"正常指令竟达 0.78** | **灾难性漂移**。0.5B 模型无法区分"逻辑谬误"与"复杂指令"，只要句子稍长（含嵌套/代码关键词），浅层注意力必涣散。 |
| **范数扫描 (Norm)** | 大段重复/废话在 128 chunk 下呈现明显信号洼地 | 128 下完全淹没；**只有缩到 32 才能看** | **尺度漂移**。256 个 Token 的平均池化抹平了一切物理波动，导致该指标无法泛化到真实 RAG 的常规切块策略（通常 256-512）。 |
| **开箱即用性** | 0 参数，默认阈值通用 | **必须针对文本风格（叙事/代码/指令）手动调参** | **最致命漂移**。失去了 Linter 的"一键体检"灵魂，变成工程师的"手调游标卡尺"。 |

### 3. 根本原因（为什么物理量失效了？）
*   **代码/指令的"任务复杂度"伪装成了"逻辑混乱"**：0.5B 的浅层网络只认得词性（主谓宾），它分不清"出于恶意的撒谎"和"出于严谨的复杂条件判断"。因此，"写快排"这种人类看来清晰明确的指令，在小模型看来是"一大堆动词和变量名的强混乱组合"，熵值自然飙升。**信噪比被任务难度污染，而非被文本质量污染**。
*   **平均池化的"信息黑洞"**：128 个 Token 的 L2 范数取平均，等价于让一个视力正常人退后 20 米去读书。只要段落内部有起伏，平均后就只剩白噪声。

### 4. 项目可复用资产（不是白干）
尽管路线终止，但本次实验产出了以下**高价值工程脚手架**，建议归档至内部 `ai-infra/experiments/` 目录：
1.  **轻量级 Hook 提取器**：能秒级提取 0.5B 模型 `attentions` 与 `hidden_states` 的稳定工程管道，可供后续"模型可解释性"其他课题复用。
2.  **负向基准数据集**：本次 86 个测试用例（含代码指令、叙事歧义、RAG 长文）已标注"人类易读性"与"模型物理量"对照，可作为团队未来验证"输入敏感度"的静态测试集。
3.  **阈值非泛化性结论**：用实锤数据堵死了团队内部未来可能提出的"是否要投入 GPU 做代理模型蒸馏"的方向，**节省了潜在 3-6 个月的研发成本**。

---

### 5. 最终意见（签字确认）
> "本次实验以极小代价（1.5 小时人力 + 0.5B 模型推理），验证了'基于浅层物理量的静态 Linter'在工业级自然语言/代码混合输入下不可行。**学术有价值，产品无出路**。建议正式终止该方向，后续 RAG 优化回归 Rerank 与 Embedding 微调主线。"
