## Test Seam
Primary test seam: `PromptLinter.analyze()` — 模块级 API 测试
Mock strategy: mock model.forward()（固定 attention 矩阵 + hidden states）+ 真实 tokenizer

## Key Interactions to Verify

### Input Validation
- Empty input `analyze("")` → InputEmptyError
- Whitespace-only `analyze("   ")` → InputEmptyError
- Very short input `analyze("a")` → safe fallback, all risk_level = "low"
- UI should warn when input < 20 characters

### Input Length
- Input > model_max_length → InputTooLongError or silent truncation with UI notice
- Truncation notice mentions "仅分析前 N tokens"

### Core Analysis (with mocked model)
- Normal input → result dict contains `token_risks`, `chunk_risks`, `metadata`
- `token_risks[].keys()` = token, token_id, entropy_delta, risk_level
- `chunk_risks[].keys()` = chunk_index, start_token, end_token, text_snippet, norm_score, is_weak
- `metadata.keys()` = model_name, total_tokens, analysis_time_ms
- When deep entropy > shallow entropy → some tokens have risk_level = "high"
- When deep entropy ≈ shallow entropy → all risk_level = "low"

### Norm Scanning (with mocked model)
- Normal text → chunks are correctly sized (chunk_size parameter respected)
- Custom chunk_size overrides default
- When a chunk has significantly lower norm → is_weak = True
- When all chunks have similar norm → is_weak = False for all

### Model Lifecycle
- `is_ready` initially False
- `unload_model()` does not raise when model not loaded
- `run_ablation_on_70B()` → NotImplementedError

### Separate API Methods
- `analyze_entropy("")` → InputEmptyError
- `scan_norm("")` → InputEmptyError

## Edge Cases
- Multi-byte text (Chinese, emoji) — tokenization and preview should not crash
- Repeated same input → cached result (no second forward pass)
- Model load failure → graceful error message, not stack trace
- Input length exactly at model_max_length → should succeed
- Input length exactly at chunk_size boundary → correct number of chunks

## Critical Paths
1. User pastes short prompt → clicks analyze → sees entropy heatmap + highlighted tokens
2. User pastes long text → clicks scan → sees chunk weakness diagram + weak chunk details
3. First launch → model downloads → progress feedback → analysis works
4. Second analysis with same text → instant result from cache
